# <a name="microsoft-project-service-description"></a>Descripción del servicio de Microsoft Project

## <a name="overview"></a>Información general

Microsoft Project ofrece las siguientes aplicaciones para ayudar a satisfacer las necesidades de su organización en cuanto a gestión de proyectos y trabajos:

- Project para la web
- Project Online 
- cliente de escritorio Project Online

Este artículo le ayudará a comprender qué aplicaciones se proporcionan en cada suscripción Project y qué capacidades ofrece cada aplicación.

### <a name="project-for-the-web"></a>Project para la web

Project para la web es la oferta más reciente de Microsoft para la administración de proyectos y trabajo basada en la nube. Project para la web proporciona capacidades de gestión de trabajo sencillas y potentes para satisfacer la mayoría de las necesidades y roles. Project gerentes y miembros del equipo pueden usar Project para que la web planifique y administre el trabajo de cualquier tamaño.

Project para la web se basa en Microsoft Power Platform. Power Platform consta de PowerApps, Power Automate, Power BI y Microsoft Dataverse. Project para los datos web se almacena en Microsoft Dataverse. 

Project para la web está disponible a través de tres suscripciones: Project Plan 1, Project Plan 3 y Project Plan 5. Las características enumeradas en [Project para las características web](#project-for-the-web-features) interactúan solo con los datos de la Project para el almacén de datos web.

### <a name="project-online"></a>Project Online
 
Project Online es una solución en línea flexible para Project Gestión de Carteras (PPM) y el trabajo diario. Project Online proporciona potentes capacidades de gestión de proyectos para planificar, priorizar y administrar proyectos y inversiones en carteras de proyectos &mdash; desde casi cualquier lugar de casi cualquier dispositivo. Project Online lo pueden usar gestores, administradores y visores de carteras, administradores de recursos y proyectos, y jefes y miembros de equipo.

Project Online se basa en la plataforma SharePoint y almacena datos en el almacén de datos SharePoint.

Project Online está disponible a través de dos suscripciones: Project Plan 3 y Project Plan 5. Las características enumeradas en [Project Online características](#project-online-features) interactúan solo con los datos del almacén de datos Project Online SharePoint.

La funcionalidad de miembro del equipo para Project Online está disponible a través de una suscripción Project Online Essentials o Project Plan 1. Project Online Essentials es una suscripción *de complemento de* miembro del equipo para clientes que tienen suscripciones Project Plan 3 o Project Plan 5. Project Online Essentials y Project Plan 1 suscripciones proporciona a los miembros del equipo una interfaz web para realizar actualizaciones de tareas y tiempos de los proyectos administrados en Project Online.

Los miembros del equipo con suscripciones Project Online Essentials o Project Plan 1 pueden realizar las siguientes funciones de Project Online:

- Utilice una interfaz basada en web
- Actualizar tareas, problemas y riesgos
- Enviar partes de horas
- Comparta documentos y colabore con usuarios de Microsoft Teams o Skype Empresarial

Project Online Essentials no incluye Project para la web. Project Plan 1 incluye tanto Project para la web como para Project Online Essentials.

### <a name="project-online-desktop-client"></a>cliente de escritorio Project Online

Microsoft Project Online cliente de escritorio es un programa de administración de proyectos que cuenta con programación automatizada, administración de recursos de proyectos e informes integrados. Project Online cliente de escritorio se puede utilizar como una aplicación independiente o puede conectarse a Project Online. Project Online cliente de escritorio es la versión de suscripción del cliente de escritorio Project Profesional y está disponible en suscripciones de Project Plan 3 y Project Plan 5.

### <a name="using-project-for-the-web-and-project-online-together"></a>Uso de Project para la web y Project Online juntos

Aunque Project para la web y Project Online son aplicaciones independientes, puede usarlas una al lado de la otra. Puede ver todos sus proyectos desde ambas aplicaciones en Project Home. Puede usar la capacidad Project hoja de ruta para crear hojas de ruta que incluyan proyectos de ambas aplicaciones. Además, puede usar Power BI y la Project para el paquete de contenido de Power BI web para obtener información sobre proyectos y recursos en las dos aplicaciones. <sup>3</sup>

Este diagrama muestra cómo las aplicaciones pueden trabajar juntas una al lado de la otra.

![Diagrama que muestra Project en la web y Project Online juntos](../media/project-online-applications.png)

En el futuro, Microsoft se centrará en Project para las innovaciones web. Sin embargo, puede seguir utilizando Project Online con confianza, ya que seguirá recibiendo mejoras clave de rendimiento y seguridad.

## <a name="service-considerations"></a>Consideraciones de servicio

> [!NOTE]
> Project para la web no está disponible actualmente para GCC, GCC High y DoD. Estamos trabajando en la entrega de Project para la web para usted, pero no podemos compartir un plazo firme en este momento.

### <a name="licensing-considerations"></a>Consideraciones sobre licencias

- Cualquier interacción en un sitio de Project Online requiere al menos una suscripción Project Plan 3 o Project Plan 5 dentro del inquilino.
- Cuando caduque la última suscripción Project Plan 1, Project Plan 3 o Project Plan 5, el Project de las instancias web no se eliminará automáticamente hasta que no tenga suscripciones activas que dependan del Microsoft Dataverse.
- Cuando caduque la última suscripción Project Plan 3 o Project Plan 5, las instancias de Project Online se eliminarán después de 120 días.
- Para Project Online suscripciones de prueba, las instancias de prueba se eliminarán 30 días después de que finalice el período de prueba.
- Para Project para las suscripciones de prueba web, las instancias de prueba no se eliminarán hasta que no tenga suscripciones activas que dependan del Microsoft Dataverse.

### <a name="use-of-sharepoint-online"></a>Uso de SharePoint Online

Project Online requiere el uso de SharePoint Online, que se aprovisiona como parte de Project Online. Los derechos sobre la funcionalidad SharePoint Online proporcionada con suscripciones Project Plan 3 o Project Plan 5 se limitan a almacenar y acceder a datos para admitir Project Online.

### <a name="project-roadmap-and-power-automate"></a>Project Hoja de ruta y Power Automate

Project La hoja de ruta requiere el uso de Power Automate, que se aprovisiona como parte de la suscripción Project. Los derechos a Power Automate funcionalidad se limitan a las capacidades Power Automate requeridas por Project Hoja de ruta. Power Automate funcionalidad requerida por Project Hoja de ruta aparece como la **integración de datos para Project con Flow** plan de servicio en la sección **Aplicaciones** del centro de administración de Microsoft 365.

### <a name="project-for-the-web-and-microsoft-dataverse"></a>Project para la web y Microsoft Dataverse

Project para el web requiere el uso de Microsoft Dataverse para almacenar sus datos. Una base de datos de Microsoft Dataverse se aprovisiona como parte de la suscripción Project. Los derechos sobre la funcionalidad de Microsoft Dataverse se limitan a almacenar y acceder a datos para admitir Project para la web. La funcionalidad de Microsoft Dataverse requerida por Project aparece como el **servicio de datos común para Project** plan de servicio en la sección **Aplicaciones** del centro de administración de Microsoft 365.

Para Project clientes con cinco (5) o más Project para las licencias web, Project para los clientes web pueden implementarse en entornos de producción y espacio aislado de Power Platform.
